- Discord server (active as of May 2026)

## What Doesn't Exist Yet

- Content creation pipeline
- Verifier/moderation system
- Search infrastructure (using DuckDuckGo placeholder)
- Certification system
- Governance voting mechanism
- Legal entity (501c3 not filed)
- Funding

## Suggested Restart Path

### Phase 1: Exist (1-2 months)
1. Landing page with ripple design
2. Curated links to existing educational resources
3. Simple user submission form
4. Host on GitHub Pages or Vercel (free)

### Phase 2: Content (3-6 months)
1. Markdown-based guide system
2. YAML prerequisite tagging
3. Basic hierarchy/navigation
4. Community contribution via GitHub PRs

### Phase 3: Quality (6-12 months)
1. Verifier review system
2. User feedback/ratings
3. Content improvement workflow

### Phase 4: Ecosystem (1-2 years)
1. Certification (if philosophically resolved)
2. Advanced search
3. Mobile app
4. Institutional partnerships

## Key Decisions to Make

1. **Content strategy:** Create, curate, or search? (Team leaned toward hybrid)
2. **Tech stack:** Static first or dynamic from start?
3. **Certification:** Test-based, work-based, or none?
4. **Governance:** Who decides when the community disagrees?
5. **Funding:** Donations, grants, ads, or affiliate links?
